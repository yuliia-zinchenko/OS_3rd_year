**Варіант:** Перенаправлений в/в, Java, поєднання future
**ОпераціїЖ** Сума дійсних


Програма розбиває обчислення математичної функції на дві незалежні частини, які виконуються паралельно в окремих процесах операційної системи.

Головний процес Manager керує запуском, передачею даних, моніторингом стану та агрегацією результатів від дочірніх процесів Workers.


Система обчислює значення функції:
$$F(x) = (x^2) + (2x)$$

Обчислення декомпозовано на два незалежних воркера:
1.  **Worker 0 (`fn1`):** Обчислює $f_1(x) = x^2$
2.  **Worker 1 (`fn2`):** Обчислює $f_2(x) = 2x$
3.  **Manager:** Агрегує результат $Result = f_1(x) + f_2(x)$

**Кроки:**

Ініціалізація: Manager зчитує число x від користувача.

Запуск процесів: Створюються два екземпляри JVM для класу Worker з різними аргументами.

Передача даних (IPC): Manager записує x у OutputStream кожного процесу та закриває потік (надсилає EOF), сигналізуючи про завершення передачі.

**Асинхронне виконання:**

Воркери виконують обчислення зі штучними затримками (Thread.sleep).

Manager запускає CompletableFuture.supplyAsync для очікування результатів, щоб не блокувати головний цикл.

Моніторинг: Головний цикл Manager перевіряє три умови:

Чи готові результати (isDone).

Чи вийшов час очікування (Timeout).

Чи натиснув користувач клавішу (Manual Interruption).

Агрегація: При успішному завершенні результати сумуються.


Якщо ввести невелике число програма завершить роботу за ~3 секунди (час найдовшого воркера).


Для тестування Timeout (Soft Timeout) ввести число більше 100.

Логіка: Worker 2 увімкне затримку 15 секунд. Ліміт менеджера — 5 секунд.
